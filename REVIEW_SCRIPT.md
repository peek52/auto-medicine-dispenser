# 🎬 สคริปต์ถ่ายคลิปรีวิว — เครื่องจ่ายยาอัตโนมัติ ESP32-P4

> ใช้เป็น guide ระหว่างถ่ายทำ — มีลำดับฉาก, สิ่งที่ต้องทำหน้ากล้อง, และคำพูด (voice-over)
> ความยาวเป้าหมาย ~12-15 นาที | จำนวนฉาก 11 ฉาก

---

## ⚙️ เตรียมก่อนถ่าย

| รายการ | สถานะที่ควรเป็น |
|---|---|
| โมดูล 6 ช่อง | ใส่ยาจำลอง (เม็ดวิตามินก็ได้) อย่างน้อย 3 ช่อง |
| Wi-Fi | เชื่อมแล้ว (จอแสดงสถานะ Online) |
| Telegram bot | ผูกกับ chat แล้ว ใช้ได้จริง |
| NETPIE Dashboard | เปิดบน laptop/มือถือเตรียมไว้ |
| Web /tech | login พร้อม |
| กล้องถ่ายคลิป | ตั้ง angle หลัก = หน้าตู้ตรง 1 มุม + มือถือถ่าย screen |

## 🎬 ฉากที่ 1 — Hook + แนะนำเครื่อง (0:00 – 0:40)

**ภาพ:** Pan-up จากฐานเครื่องขึ้นมาที่หน้าจอ standby

**Voice-over (TH):**
> "นี่คือเครื่องจ่ายยาอัตโนมัติที่เราพัฒนาขึ้นเอง ใช้ ESP32-P4 Nano เป็นสมอง รองรับยา 6 ชนิด ตั้งเวลาได้ 7 มื้อต่อวัน เชื่อมต่อ NETPIE cloud และ Telegram bot — ดูแลผู้สูงอายุได้แม้อยู่คนละบ้าน วันนี้จะพาทุกคนรีวิวการใช้งานครบทุกฟังก์ชั่น"

**ข้อความบนจอ (caption):** "Automatic Pill Dispenser — ESP32-P4 + NETPIE + Telegram"

## 🎬 ฉากที่ 2 — ทัวร์ฮาร์ดแวร์ (0:40 – 1:40)

**ภาพ:** ค่อยๆ ชี้แต่ละส่วน พร้อม close-up

| Beat | ภาพ | คำพูด |
|---|---|---|
| 0:40 | เปิดประตูตู้ → ชี้ตลับ 6 ช่อง | "หกตลับแยกอิสระ บรรจุยาได้ตลับละ 16 เม็ด" |
| 0:55 | ชี้ servo MG90S | "เซอร์โว MG90S 6 ตัว ขับด้วย PCA9685 ผ่าน I2C" |
| 1:05 | ชี้กล้อง OV5647 | "กล้อง MIPI-CSI 5 ล้านพิกเซล ถ่ายภาพยืนยันการจ่ายยา" |
| 1:15 | ชี้จอ TFT 4 นิ้ว | "จอสัมผัส ST7796S ความละเอียด 480×320" |
| 1:25 | ชี้ลำโพง | "ลำโพง 8Ω 20W ขับด้วย DY-HV20T เสียงพูดภาษาไทย" |
| 1:35 | ชี้ตู้คอนโทรล | "สวิตช์ชิ่ง 5V 10A + 12V 2A แยกราง" |

## 🎬 ฉากที่ 3 — Boot sequence (1:40 – 2:30)

**ภาพ:** ปิด-เปิดเครื่อง ถ่ายตั้งแต่จอดับ → splash → standby

**สิ่งที่ผู้ชมต้องเห็น:**
1. จอดำ → splash "เครื่องจ่ายยา / อัตโนมัติ" (~2 วินาที)
2. เปลี่ยนเป็นหน้า standby (เวลา + ตารางถัดไป)
3. ได้ยินเสียง servo park ช่วงสั้นๆ ขณะใช้หน้า standby แล้ว
4. แตะหน้าจอเทสต์ — touch ตอบรับทันที

**Voice-over:**
> "เปิดเครื่องแค่ 2 วินาทีก็พร้อมใช้งาน servo ทุกตัว park ไป home อัตโนมัติในพื้นหลัง โดยที่ touch screen ใช้งานได้ทันที"

---

## 🎬 ฉากที่ 4 — Touch UI: เซ็ตอัพยา (2:30 – 4:30)

**ขั้นตอนทำหน้ากล้อง:**
1. แตะมุมล่างขวาเพื่อเข้า Menu
2. เลือก "ตั้งค่ายา" (Setup Meds)
3. แตะโมดูล 1 → เข้าหน้า detail
4. ตั้งชื่อยา → กด "ชื่อยา" → keyboard → พิมพ์ "Paracetamol" → Enter
5. ปรับจำนวน → กด + จนถึง 10
6. เลือกมื้อ → แตะ "ก่อนอาหารเช้า" + "ก่อนอาหารกลางวัน"
7. กด Save → กลับหน้า Setup Meds → เห็นโมดูล 1 มีข้อมูลครบ

**Voice-over:**
> "ตั้งค่ายาแต่ละช่องได้ง่ายๆ ผ่านหน้าจอสัมผัส กำหนดชื่อยา จำนวนเม็ด และมื้อที่ต้องจ่ายได้ 7 มื้อ — เช้า กลางวัน เย็น ก่อนนอน และก่อน-หลังอาหาร"

**Caption:** "Touch UI: ตั้งค่ายา 6 ตลับ × 7 มื้อ"

---

## 🎬 ฉากที่ 5 — NETPIE Dashboard (4:30 – 6:30)

**ภาพ:** ถ่ายมือถือ/laptop เปิด NETPIE widget

**ขั้นตอน:**
1. เปิด NETPIE freeboard ที่ตั้งไว้
2. **Widget 1 - Dashboard หลัก:**
   - แสดงสถานะ Online + เวลาปัจจุบัน
   - กราฟยาคงเหลือ 6 ช่อง
3. **Widget 2 - ตั้งเวลา:**
   - กดปุ่ม Edit → ปรับเวลามื้อใหม่ → Save
   - บนเครื่อง popup confirm เด้งขึ้น → กดอนุมัติ → Telegram ส่งสรุปการเปลี่ยนแปลง
4. **Widget 3 - Manual Dispense:**
   - เลือกโมดูล 1 → จำนวน 2 → กดจ่าย → ดู servo หมุนผ่านกล้องสด
5. **Widget 4 - คืนยาทั้งหมด:**
   - กด "คืนยาทั้งหมด" → confirm → ดู servo ทุกตัวทำงาน

**Voice-over:**
> "NETPIE Dashboard ให้คุณดูสถานะแบบ realtime ตั้งเวลาใหม่ สั่งจ่ายยาด้วยมือ หรือล้างยาทั้งระบบ — ที่สำคัญทุกการเปลี่ยนแปลงต้องผ่านการยืนยันจากหน้าจอเครื่อง ป้องกันการแก้ข้อมูลโดยไม่ตั้งใจ"

---

## 🎬 ฉากที่ 6 — Telegram Bot (6:30 – 8:30)

**ภาพ:** ถ่ายมือถือพร้อม Telegram

**ลำดับคำสั่งที่จะแสดง:**

| คำสั่ง | สิ่งที่ควรเห็น |
|---|---|
| `/status` | bot ตอบสถานะตลับ 6 ช่อง พร้อมจำนวนคงเหลือ |
| `/photo` | bot ส่งภาพ realtime จากกล้องในเครื่อง |
| `/dispense 1 2` | bot ยืนยัน → servo จ่าย 2 เม็ดจากตลับ 1 + รูปภาพหลังจ่าย |
| `/help` | bot แสดงรายการคำสั่งทั้งหมด |

**Voice-over:**
> "Telegram bot รองรับคำสั่งครบ — สั่งจ่ายยา ดูภาพสด ตรวจสถานะ และยังส่งภาพถ่ายเป็นหลักฐานทุกครั้งที่จ่ายยา ผู้ดูแลตรวจสอบจากที่ไหนก็ได้ในโลก"

**Caption:** "Telegram: /status /photo /dispense /help"

---

## 🎬 ฉากที่ 7 — Auto Dispense + Confirm popup (8:30 – 10:30)

**ภาพ:** ตั้งเวลา test ให้ตรงกับเวลาถ่าย เช่น ก่อนถ่าย 1 นาที

**ลำดับ:**
1. ถึงเวลา → จอเปลี่ยนเป็น "ถึงเวลาทานยา" สีแดง
2. ลำโพงพูด "ถึงเวลาทานยาก่อนอาหารกลางวัน"
3. **แตะหน้าจอ** (ใช้ปลายนิ้วชัดๆ) → popup หาย → "กำลังจ่ายยา"
4. servo หมุน → ยาตกลงถาด
5. จอเปลี่ยนเป็น "จ่ายยาเรียบร้อย"
6. ลำโพงพูด "จ่ายยาเรียบร้อย"
7. Telegram ได้รับภาพ + ข้อความ "จ่ายยาแล้ว มื้อ ก่อนอาหารกลางวัน — Paracetamol 2 เม็ด"

**Voice-over:**
> "ถึงเวลาแล้วเครื่องจะเตือนทั้งจอและเสียง ผู้ใช้แตะเพื่อยืนยันรับยา — เครื่องจะจ่ายยาออกมาแล้วถ่ายภาพส่งให้ผู้ดูแลทันที"

## 🎬 ฉากที่ 8 — Edge case: ยาหมด popup (10:30 – 11:30)

**ภาพ:** ตั้งโมดูลที่มี slot แต่ count=0 → รอ slot fire

**สิ่งที่ผู้ชมเห็น:**
1. ถึงเวลา → popup สีแดง "โมดูลที่ N ยังไม่ใส่ยา"
2. แสดง 2 ปุ่ม: **ล้างช่อง** (สีแดง ซ้าย) / **ใส่จำนวนยา** (สีเขียว ขวา)
3. กด "ใส่จำนวนยา" → เข้าหน้า detail ของโมดูลนั้นๆ
4. กลับมา demo "ล้างช่อง" → โมดูลถูก reset

**Voice-over:**
> "ถ้าตลับไหนตั้งค่ามื้อแต่ยังไม่ใส่ยา ระบบจะแจ้งเตือนพร้อมให้เลือกล้างค่าหรือเติมยาทันที"

## 🎬 ฉากที่ 9 — Web /tech panel (11:30 – 12:30)

**ภาพ:** เปิด browser → IP ของเครื่อง

**ลำดับ:**
1. หน้า Login → กรอกรหัส tech → Enter
2. แสดงหน้า /tech → Dashboard
3. **System tab:** แสดง max_pills, quiet hours, sound
4. **Module tab:** สลับโมดูล logical → physical
5. **Camera tab:** ดู MJPEG stream live + ปุ่ม snapshot
6. **Servo tab:** ตั้ง home/work angle ของแต่ละ servo
7. **Audio test:** กดปุ่มทดสอบเสียง

**Voice-over:**
> "หน้า /tech สำหรับช่างเทคนิค ตั้งค่าระดับ system ดู camera stream ปรับ servo angle และทดสอบเสียง — รหัสเข้าใช้แยกจากผู้ใช้ทั่วไป"

## 🎬 ฉากที่ 10 — Hardware fail alert (12:30 – 13:30)

**ภาพ:** ถอดสาย PCA9685 ออก → reboot

**สิ่งที่ผู้ชมเห็น:**
1. Boot ผ่านปกติ
2. หลัง 3 นาที popup สีแดงเด้ง "ฮาร์ดแวร์ผิดปกติ — กรุณาปิดเปิดเครื่อง"
3. Telegram ได้รับข้อความเตือน (พร้อม rate-limit 1 ชม.)
4. แตะหน้าจอที่ไหนก็ปิด popup ได้

**Voice-over:**
> "ถ้าฮาร์ดแวร์มีปัญหา เครื่องจะแจ้งทั้งบนจอและส่ง Telegram ให้ผู้ดูแลรู้ทันที — ไม่ต้องนั่งเฝ้าตู้ตลอด"

**ภาพ:**

2. รอ slot fire → ยังจ่ายยาได้
3. เสียบ Wi-Fi กลับ → Telegram ค้างคิวจะปล่อยทีเดียว

**Voice-over (สรุป):**

## 🎙️ Tip การถ่าย

1. **เสียง:** ใช้ไมค์ติดเสื้อหรือ shotgun mic — ลำโพงเครื่องเสียงดัง ระวังตัดทับเสียงพูด
2. **แสง:** ไฟวงแหวนส่องหน้าจอตรงๆ จะทำให้จอสะท้อน — ใช้ไฟด้านข้าง 45°
3. **มุมกล้องจอเครื่อง:** กล้องทำมุม 5-10° กับจอ ป้องกัน moire pattern
4. **ตัดต่อ:**
   - ใส่ caption thai-english คู่กัน (อาจารย์ต่างชาติเข้าใจง่าย)
   - timestamp ฉาก: 0:00, 1:40, 4:30, ฯลฯ ใน description
   - speed-up 2x ในฉากตั้งค่า (ไม่ต้องดูทุก keystroke)
5. **คำลงท้าย:** "ขอบคุณที่ดูจนจบ — ฝากกด subscribe ครับ"
