1.1 ความเป็นมาและความสำคัญของปัญหา                       (~3 หน้า)
    1.1.1 สถานการณ์ผู้สูงอายุและผู้ป่วยโรคเรื้อรังในประเทศไทย
          - สถิติประชากรผู้สูงอายุ (กรมกิจการผู้สูงอายุ)
          - สัดส่วนผู้ป่วยโรค NCDs ที่ต้องทานยาประจำ (กรมควบคุมโรค)
          - แนวโน้มการเพิ่มขึ้นของผู้สูงอายุที่อยู่บ้านคนเดียว
    1.1.2 ปัญหาการลืมรับประทานยา (Medication Non-adherence)
          - นิยาม WHO: adherence rate ≤ 50% ในผู้สูงอายุ
          - ผลทางคลินิก: รักษาไม่หาย, ดื้อยา, ผลข้างเคียงจาก overdose
          - ผลทางเศรษฐกิจ: ค่ารักษาพยาบาลเพิ่ม, การกลับเข้าโรงพยาบาล
    1.1.3 ข้อจำกัดของเครื่องจ่ายยาที่มีในตลาด
          - แบบ mechanical: ไม่มีการแจ้งเตือนระยะไกล
          - แบบ smart ราคาแพง: 10,000-30,000 บาท/เครื่อง
          - ส่วนใหญ่ไม่มีกล้องตรวจสอบ ไม่มี cloud sync
    1.1.4 แนวโน้มเทคโนโลยี IoT + Healthcare ในปี 2026
          - การลดราคา ESP32 family
          - NETPIE 2020 platform ของไทย
          - Telegram Bot API ที่เข้าถึงง่าย

2.1 บทนำของบทที่ 2                                         (~1 หน้า)

2.2 สถาปัตยกรรมไมโครคอนโทรลเลอร์ ESP32-P4                  (~3-4 หน้า)
    2.2.1 RISC-V dual-core architecture
          - HiFi 5 vs RISC-V (เปรียบเทียบกับ ESP32 classic)
          - 360 MHz หลัก + 40 MHz LP-core
          - Memory map: 768 KB internal SRAM + 32 MB PSRAM
    2.2.2 Hardware accelerators
          - JPEG codec (encode/decode)
          - PPA (Pixel Processing Accelerator)
          - 2D-DMA + EDMA
    2.2.3 I/O peripherals
          - 55+ GPIO
          - 2× I²C, 5× SPI, 6× UART, 2× SDIO
          - MIPI-CSI 2-lane, MIPI-DSI

2.3 ESP-IDF Framework และ FreeRTOS                         (~3 หน้า)
    2.3.1 ESP-IDF architecture
          - Component-based build system (CMake)
          - Driver layer + HAL + LL layer
          - Managed components
    2.3.2 FreeRTOS principles
          - Task scheduling (preemptive priority-based)
          - Semaphore, Mutex, Queue
          - Tick rate, Watchdog
    2.3.3 IDLE0/IDLE1, TWDT, INT_WDT

2.4 โปรโตคอลการสื่อสารฮาร์ดแวร์                          (~5-6 หน้า)
    2.4.1 I²C Protocol
          - Master-slave, 7-bit addressing
          - Start, Address, R/W, ACK, Data, Stop
          - Pull-up resistor calculation: R = (Vcc-VOL) / I_OL
                ตัวอย่าง: 3.3V, 3 mA → R = 1.1 kΩ
                Bus capacitance budget Cb ≤ 400 pF (Standard)
          - Speed modes: Standard 100 kHz, Fast 400 kHz
    2.4.2 I²C Multiplexer (TCA9548A)
          - 8-channel switch กับ control byte
          - การแก้ปัญหา address collision (VL53 ทุกตัว @ 0x29)
    2.4.3 SPI Protocol (สำหรับจอ TFT)
          - Master mode, MSB-first, mode 0
          - Frequency 40 MHz บน ESP32-P4
    2.4.4 MIPI-CSI (Camera)
          - Differential pairs (D0±, D1±, CLK±)
          - Lane bit rate 200 Mbps × 2 lanes
          - Packet format: Short packet, Long packet
    2.4.5 UART (Audio module)
          - Asynchronous, 9600/115200 baud
          - 8-N-1 framing
    2.4.6 SDIO (WiFi co-processor)
          - 4-bit data lines, 40 MHz clock
          - Function 0 + Function 1 split

2.5 ทฤษฎีเซนเซอร์                                          (~5-6 หน้า)
    2.5.1 IR Photointerruptor (Break-beam Sensor)
          - หลักการ: LED + Phototransistor วางหันหน้ากัน
          - กระแสไหลปกติ → ยาตกตัด beam → กระแสลดลง
          - Output dilevel: HIGH ปกติ, LOW เมื่อบังลำแสง
          - Debounce theory + edge detection
    2.5.2 Time-of-Flight (VL53L0X)
          - หลักการ d = c × Δt / 2
          - VCSEL laser 940 nm
          - SPAD photodiode array
          - Ranging modes: Default, Long Range, High Speed, High Accuracy
          - การแปลง raw_mm → pill_count
          - Signal rate, range valid status
    2.5.3 Capacitive Touch (FT6336U)
          - Mutual capacitance vs Self capacitance
          - X-Y electrode matrix
          - I²C interface, register 0x02 = touch data
          - Interrupt mode vs Polling mode

2.6 ทฤษฎี Actuator                                         (~3 หน้า)
    2.6.1 Servo Motor SG90
          - 50 Hz PWM, pulse 0.5-2.5 ms
          - Position = (pulse_us - 500) × 180 / 2000 องศา
          - Torque vs current draw
    2.6.2 PCA9685 (16-channel PWM driver)
          - I²C @ 0x40, 12-bit resolution
          - Internal 25 MHz oscillator + prescaler
          - prescale = round(25e6 / (4096 × f_pwm)) - 1
            ที่ 50 Hz: prescale = 121

2.7 Computer Vision Subsystem                              (~3-4 หน้า)
    2.7.1 OV5647 image sensor
          - 5 MP CMOS (2592×1944), Bayer pattern
          - SCCB control (I²C compatible @ 0x36)
          - PID 0x5647 (sometimes 0x568f variant)
    2.7.2 ISP Pipeline บน ESP32-P4
          - Bayer demosaic → RGB → YUV422
          - Auto exposure, Auto white balance
          - Output bit depth 8-bit per channel
    2.7.3 JPEG Hardware Encoder
          - Block-based DCT (8×8)
          - Quantization tables
          - Huffman coding
          - Throughput: ~50 MB/s output
    2.7.4 Software 90° rotation algorithm
          - dst(x,y) = src(y, src_w-1-x)
          - YUV422: Y0 U Y1 V → 4 bytes per 2 pixels
          - Buffer requirement: PSRAM 1 MB

2.8 IoT และ Cloud Communication                            (~5-6 หน้า)
    2.8.1 MQTT Protocol
          - Publish-Subscribe pattern
          - QoS 0, 1, 2 differences
          - Connect, Subscribe, Publish, PingReq frames
          - Last Will Testament
    2.8.2 NETPIE 2020 Platform (ของไทย)
          - Architecture: Device → MQTT broker (mqtt.netpie.io:1883)
          - Device Shadow concept (synchronized state)
          - Topic: @shadow/data/update, @msg/...
          - Authentication: Device Token
    2.8.3 HTTPS + TLS 1.2
          - X.509 certificate chain
          - esp-x509-crt-bundle (Mozilla CA bundle)
          - Handshake: ClientHello → ServerHello → KeyExchange → Finished
    2.8.4 Telegram Bot API
          - HTTPS REST API: api.telegram.org
          - getUpdates long-polling (offset-based)
          - sendMessage, sendPhoto (multipart/form-data)
          - File upload limit: 50 MB
    2.8.5 Google Apps Script Webhook
          - Web App URL ที่ deploy เป็น public
          - POST JSON → Sheets row
          - Quota: 6 นาที/execution

2.9 Algorithm และเทคนิคซอฟต์แวร์                          (~4-5 หน้า)
    2.9.1 Edge-Triggered Detection with Debouncing
          - State machine: IDLE → RISING → STABLE → FALLING → IDLE
          - Debounce window: 30 ms minimum stable time
    2.9.2 Stable-Tick Filter (Sensor → Shadow Sync)
          - Skip publish until reading stable for N ticks
          - ลด MQTT traffic + flicker
    2.9.3 Mid-Cycle Bail (Dispense Timeout)
          - servo work command + IR poll loop
          - bail หลัง 4.5s ถ้าไม่เห็นยา
    2.9.4 Three-Tier Watchdog
          - Tier 1: Task WDT (TWDT) 30s — task hang
          - Tier 2: Liveness canary task — software lockup
          - Tier 3: RTC WDT — full firmware crash
    2.9.5 Mutex Coordination Pattern
          - g_i2c_mutex (single owner)
          - Pause/resume API (cooperative yield)
          - Avoid priority inversion
    2.9.6 Lazy Initialization
          - Camera + JPEG encoder on first use
          - Saves ~3s boot time, ~1MB PSRAM idle
    2.9.7 EMA Filter (Exponential Moving Average)
          - filtered = (filtered × (1-α)) + (raw × α)
          - α = 0.6 → fast tracking, ลด noise

2.10 Security and Reliability                              (~2 หน้า)
    2.10.1 Defense-in-depth (NVS encryption disabled, fall-back code)
    2.10.2 Offline-first design (queue + retry)
    2.10.3 Hardware bus recovery (bit-bang SDA unstick)

    3.5.1 Pill Dispense Flow
          [Flowchart + pseudocode]
          ```
          dispense(med_idx, qty):
              acquire_busy_lock()
              for i in 1..qty:
                  pcf8574_warmup()            # 8 reads × 10 ms
                  servo_go_work(med_idx)
                  loop_start_ms = now()
                  consec_low = 0
                  while elapsed < 7.5s:
                      if elapsed > 1.5s and not home_issued:
                          servo_go_home(med_idx)
                          home_issued = True
                      ir = pcf8574_read()
                      if (ir & (1<<med_idx)) == 0:
                          consec_low += 1
                          if consec_low >= 3:
                              pill_detected = True
                      else:
                          consec_low = 0
                      yield(1 tick)
                  if pill_detected:
                      decrement_count_in_shadow()
                      send_telegram_photo()
              release_busy_lock()
          ```

4.5 ปัญหาที่พบและการแก้ไข                                  (~6-8 หน้า)
    [แทรก WORKFLOW_REPORT.html — bug timeline]
    4.5.1 ESP-IDF v5.3.2 i2c_master ISR race
          อาการ: Store Access Fault, Camera SCCB INVALID_STATE
          วิธีหา: backtrace เห็น i2c_isr_receive_handler
          วิธีแก้: HAL pointer guard (>=0x40000000)
                + state-reset retry on INVALID_STATE
          ทดสอบ: 1000 SCCB writes, 0 panics หลังแก้
    4.5.2 Power sequencing latch (Camera)
          อาการ: Camera SCCB NACK ทุก attempt หลัง cold boot
          วิธีแก้: AVDD LDO power-cycle (acquire→release→wait→reacquire)
          AVDD off-time tuning: 200ms → 800ms → success rate 50% → 95%
    4.5.3 I²C bus loading (signal integrity)
          อาการ: VL53 init fail at random register
          การวินิจฉัย: oscilloscope rise time 1.8 µs (สเปก ≤ 1 µs)
          วิธีแก้: 2.2 kΩ pull-up + per-op retry × 5 + XSHUT hard-reset
    4.5.4 Phantom Touch
          อาการ: standby กระโดดไปเมนูเอง
          วิธีแก้: PRESS_DEBOUNCE=2 + COORD_STABILITY 150px + boot mute 3s
    4.5.5 Late-detect re-bootstrap loop
          อาการ: VL53 re-init ~13s หลัง boot → I²C bus ค้าง 10s
          วิธีแก้: cache `tca_was_up_at_boot` flag (ไม่ re-ping)
    4.5.6 Telegram strips EXIF orientation
          อาการ: ภาพ Telegram แนวนอน (ผิด orientation)
          วิธีแก้: software pixel rotation YUV422 90° CCW
    4.5.7 Touch freeze after long use
          อาการ: จอค้างหลังใช้ ~ชั่วโมง
          วิธีแก้: auto-recovery (8 fail → CTP_RST + bus recovery)
                cool-down 2 วินาที (เดิม 30s)
    4.5.8 Slot button flicker
          อาการ: ปุ่ม ก่อน/หลัง กระพิบตอนกด
          วิธีแก้: per-button partial redraw (เดิม redraw ทั้ง panel)
