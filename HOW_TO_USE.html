1แนะนำเครื่อง

เครื่องจ่ายยาอัตโนมัติออกแบบมาเพื่อช่วยผู้ป่วยและผู้สูงอายุที่ต้องทานยาตามมื้อ เพื่อความปลอดภัยและความสะดวกในการรับยาในแต่ละมื้อ

คุณสมบัติเด่น

  • 🕐 ตั้งเวลาแยกได้ 7 มื้อ/วัน: เช้า (ก่อน/หลัง), กลางวัน (ก่อน/หลัง), เย็น (ก่อน/หลัง), ก่อนนอน
  • 💊 รองรับยาได้สูงสุด 6 ตลับ ต่อโมดูล
  • 📱 ใช้งานทั้งผ่าน หน้าจอสัมผัส บนเครื่อง และ คลาวด์ NETPIE
  • 📷 มี กล้อง ถ่ายรูปยืนยันการรับยา ส่งเข้าแอป Telegram
  • 🔔 แจ้งเตือนเสียงล่วงหน้า 30/15/5 นาที + เสียงปลุกเมื่อถึงเวลา
  • ⚡ ทำงาน ออฟไลน์ได้เต็มรูปแบบ — ไม่ต้องอยู่ติดอินเตอร์เน็ตตลอดเวลา

ส่วนประกอบหลัก

ส่วนหน้าที่
หน้าจอสัมผัส 4.3"ตั้งค่า, ดูเวลา, รับยาด้วยการกดยืนยัน
ตลับยา 6 ช่องเก็บเม็ดยาแยกประเภท สูงสุด 16 เม็ด/ตลับ
เซ็นเซอร์ IRตรวจจับเม็ดยาที่จ่ายออก
กล้อง MIPI-CSIถ่ายรูปยายืนยันก่อนส่งเข้า Telegram
ลำโพง (DFPlayer)เสียงแจ้งเตือนภาษาไทย/อังกฤษ

2เปิดเครื่องครั้งแรก

1
เสียบสาย USB Type-C 5V/3A เข้าด้านหลังเครื่อง — รอประมาณ 7-10 วินาที หน้าจอเปิด
2
เครื่องเล่นเสียงทักทาย + แสดงหน้า Standby (โมดูล 1-6 พร้อมเวลามื้อ)
3
แตะหน้าจอเพื่อเข้าสู่ เมนูหลัก (4 ปุ่ม: ตารางยา, จัดการตลับ, ตั้งค่าระบบ, WiFi)
💡 ข้อมูลแรกที่ต้องตั้ง
หากเปิดครั้งแรก เครื่องจะใช้ค่าเริ่มต้น (08:00, 08:30, 12:00, 12:30, 17:00, 17:30, 21:00) และทุกตลับยังว่าง — ควรตั้ง WiFi และตารางยาของผู้ป่วยก่อนใช้งานจริง

3ตั้งค่า WiFi

การเชื่อม WiFi จำเป็นถ้าต้องการใช้งานผ่านคลาวด์ NETPIE และการแจ้งเตือน Telegram ถ้าใช้แค่ออฟไลน์ข้ามขั้นตอนนี้ได้

วิธีตั้งผ่านหน้าจอเครื่อง

1
แตะ 🛜 WiFi ในเมนูหลัก
2
รอเครื่อง สแกน WiFi รอบ ๆ (~3 วินาที) แล้วเลือก SSID ที่ต้องการ
3
พิมพ์รหัสผ่านผ่านคีย์บอร์ดบนจอ แล้วกด เชื่อมต่อ
4
ถ้าสำเร็จ เครื่องจะแสดง IP Address และจดจำไว้ (ไม่ต้องตั้งใหม่ทุกครั้งที่เปิด)

วิธีตั้งผ่านเว็บ (Setup AP)

ถ้าหน้าจอตั้งไม่ได้ หรือยังไม่เคยตั้ง WiFi:

1
เปิดมือถือ → เชื่อม WiFi ชื่อ Automatic Pill Dispenser (ไม่มีรหัส)
2
เปิดเบราเซอร์ ไป http://192.168.4.1/wifi
3
เลือก WiFi + พิมพ์รหัสผ่าน → บันทึก เครื่องจะรีบูตและเชื่อม WiFi ใหม่

4ตั้งตารางเวลายา

เครื่องรองรับ 7 มื้อ แต่ละมื้อตั้งเวลาแยกอิสระ (มื้อไหนไม่ใช้ ปล่อยว่างได้)

วิธีตั้ง

1
เมนูหลัก → 📅 ตารางยา
2
แตะ การ์ดมื้อใด ๆ (เช่น "ก่อนอาหารเช้า") → เปิดตัวเลือกเวลา
3
ใช้ปุ่ม ▲ ▼ ปรับ ชม. และ นาที กดค้างปุ่มเพื่อ ปรับเร็ว
4
กด บันทึก เครื่องจะพูด "บันทึก" และกลับมาหน้าตาราง
✨ Auto-คำนวณคู่มื้อ
ตั้ง ก่อนอาหาร 07:00 → เครื่องจะคำนวณ หลังอาหาร = 07:45 ให้อัตโนมัติ (เว้น 45 นาทีเพื่อความปลอดภัยทางยา) ถ้าจะปรับเปลี่ยน กดเข้าไปที่ slot คู่นั้นและแก้ได้ แต่ ห่างกันไม่เกิน 1 ชม.
⚠️ กฎที่ระบบจะ Reject
  • เวลาก่อน ≥ เวลาหลัง (เช่น ก่อน 08:00, หลัง 07:30)
  • ห่างก่อน/หลัง เกิน 1 ชม. (เช่น ก่อน 07:00, หลัง 09:00)
  • มีเวลาเดียวกันใน 2 มื้อ

5ตั้งค่าตลับยา (โมดูล 1-6)

แต่ละตลับยาตั้งได้ 3 ค่า: ชื่อยา, มื้อที่ใช้, จำนวนยาในตลับ

1
เมนูหลัก → 💊 จัดการตลับ
2
แตะการ์ดโมดูล 1-6 ที่ต้องการ → หน้าตั้งค่าตลับ
3
กด ชื่อยา → พิมพ์ผ่านคีย์บอร์ดไทย/อังกฤษ
4
เลือก มื้อที่ใช้ (ติ๊กได้หลายมื้อ) — เช่นยาแก้แพ้ใช้ทั้งเช้าและเย็น
5
ปรับ จำนวนยาในตลับ (กด +/- หรือกดค้างเพื่อเพิ่มเร็ว)
6
กด บันทึก ที่มุมล่าง
🔄 ป๊อปอัพ "เติมยา/ล้างข้อมูล"
ถ้าเปิดตลับที่จำนวนยา = 0 (และเคยตั้งชื่อ/มื้อไว้แล้ว) เครื่องจะถามทันที:
  • เติมยา → ปิดป๊อปอัพ ให้ user ปรับจำนวนยา +
  • ล้างข้อมูล → ลบชื่อ/มื้อ/จำนวนทั้งหมดของตลับนี้

6การใช้งานประจำวัน

เมื่อถึงเวลายา

  1. เครื่องดัง เสียงปลุก + แสดงป๊อปอัพแดง "ถึงเวลาทานยา" + ส่ง Telegram
  2. เสียงปลุก ซ้ำทุก 9 วินาที จนกว่าจะกดตอบสนอง
  3. ผู้ป่วยมา แตะที่จอ เพื่อยืนยัน → servo หมุนจ่ายยา → กล้องถ่ายรูป → ส่งภาพยาเข้า Telegram

เสียงเตือนล่วงหน้า

เวลามื้อก่อนอาหารมื้อหลังอาหาร
30 นาที✅ มีมีเฉพาะถ้าก่อนอาหารคู่นั้นไม่มียา
15 นาที✅ มี✅ มี
5 นาที✅ มี✅ มี
ถึงเวลาจ่ายเสียงปลุก + Telegramเสียงปลุก + Telegram

เสียงเตือนล่วงหน้าจะตามด้วยชื่อมื้อ เช่น "อีก 15 นาที + ก่อนอาหารกลางวัน"

⏰ ถ้าไม่กดยืนยันภายใน 15 นาที
เครื่องจะข้ามมื้อนั้นโดยอัตโนมัติ — ไม่จ่ายยา และส่ง Telegram แจ้ง "❌ ไม่มีผู้มารับยา" รอแจ้งเตือนมื้อถัดไป

กรณียาหมด

ถ้าตอนถึงเวลา แต่บางตลับยาหมด (count = 0):

  • เครื่องส่ง Telegram แจ้งล่วงหน้า: "⚠️ แจ้งเตือนยาหมด - โมดูลที่ต้องเติม: [ชื่อยา]"
  • ตลับที่มียา ยังจ่ายตามปกติ
  • ตลับที่หมด ขึ้นรายงาน "ไม่พบยา"

7ควบคุมผ่านคลาวด์ NETPIE

NETPIE คือคลาวด์ MQTT ที่เครื่องเชื่อมไปอัตโนมัติ ใช้สำหรับดูสถานะและสั่งงานจากระยะไกล

Widget ที่ใช้ได้

Widgetใช้ทำอะไร
Dashboardดูสถานะตลับยา + เวลามื้อ + จำนวนยาที่เหลือ
Settingsแก้ไขชื่อยา/มื้อ/จำนวน/ตารางเวลา (รออนุมัติที่จอ)
Manual Dispenseสั่งจ่ายยาด่วน เฉพาะตลับ หรือ ทั้งระบบ
🔒 ระบบ "รออนุมัติ"
การแก้ไขข้อมูลผ่าน NETPIE จะ ไม่ commit ทันที — จะแสดงป๊อปอัพที่หน้าจอเครื่องเพื่อให้ผู้ดูแลเครื่อง กดอนุมัติก่อน + ส่ง Telegram ขอยืนยัน เพื่อป้องกันการแก้ไขโดยไม่ตั้งใจจากระยะไกล (หน้าจอเครื่อง = source of truth)

8การแจ้งเตือนผ่าน Telegram

Telegram Bot ส่งข้อความและรูปภาพให้ผู้ดูแลทราบเหตุการณ์ทั้งหมด

ประเภทข้อความที่ส่ง

  • 🔔 เตือนล่วงหน้า: "อีก 30/15/5 นาทีถึงเวลายา..."
  • ⏰ ถึงเวลาแล้ว: "ถึงเวลาทานยา มื้อ: ก่อนอาหารเช้า กรุณามารับยาที่เครื่อง"
  • ✅ จ่ายสำเร็จ: "จ่ายยาสำเร็จ" + รูปภาพยาในถาด
  • ❌ ไม่มีผู้รับยา: "ไม่มีผู้มารับยาภายใน 15 นาที ระบบจึงไม่จ่ายยาออกมา"
  • ⚠️ ยาหมด: ระบุชื่อตลับที่ต้องเติม
  • 📷 รูปยืนยันการรับยา: ส่งเข้า chat ทุกครั้งที่จ่ายสำเร็จ

การตั้งค่า

ตั้งใน /tech → Telegram โดยใส่ Bot Token + Chat ID ที่ได้จาก BotFather

9หน้าผู้ดูแล (/tech)

หน้าเว็บสำหรับช่าง/ผู้ดูแล ใช้ตรวจสอบและปรับแต่งระบบเชิงลึก

วิธีเข้า

  1. เครื่องและมือถือต้องอยู่ใน WiFi เดียวกัน
  2. เปิดเบราเซอร์ → http://[IP-ของเครื่อง]/ หรือ /tech
  3. ใส่รหัสช่าง (กำหนดจาก /access)

ฟังก์ชันในหน้า /tech

  • 📷 กล้อง: ดูสตรีมสด + ปรับ ความสว่าง/ความชัด/ความอิ่มสี (-3 ถึง +3)
  • ⚙️ Servo Test: ทดสอบหมุน servo แต่ละโมดูล
  • 🔊 เสียง: ปรับเสียงเตือน, เสียงปุ่ม, ช่วงเสียงปลุก (วินาที)
  • 🛑 Emergency Stop: หยุด servo ทันที (กรณีฉุกเฉิน)
  • 🔄 Reboot: รีบูตเครื่องผ่านเว็บ
  • 📊 Audit Log: ประวัติการจ่ายยาทุกครั้ง (เก็บใน NVS)
  • 🗑️ Factory Reset: ล้างค่าทั้งหมดกลับเป็น default

10การแก้ปัญหาเบื้องต้น

เครื่องไม่เปิด / หน้าจอดับ

  • ตรวจสาย USB-C ต้อง 5V/3A (อะแดปเตอร์ Type-C ปกติทั่วไปใช้ได้)
  • ลองสายอื่น (สายอ่อนอาจไม่ส่งกระแสพอ)
  • กดปุ่ม Reset ด้านหลังเครื่องค้าง 3 วินาที

servo ไม่หมุนเมื่อถึงเวลายา

  • เช็คว่า Emergency Stop ใน /tech ไม่ได้เปิดอยู่
  • เช็คตลับยานั้นมียา (count > 0)
  • เช็คว่ามื้อนั้นได้ติ๊กเลือกตลับนี้ไว้
  • ถ้า servo ค้าง ลองกด Servo Home ใน /tech

WiFi เชื่อมไม่ติด

  • เครื่องรองรับเฉพาะ WiFi 2.4GHz เท่านั้น ไม่รองรับ 5GHz
  • เช็ครหัสผ่านพิมพ์ถูกต้อง
  • ถ้ายังไม่ติด ลองพิมพ์ wifi clear ใน console (USB serial)
  • หรือ Factory Reset ผ่าน /tech

Telegram ไม่ส่ง

  • เช็ค WiFi ติดอยู่จริง (IP แสดงในหน้าจอ)
  • เช็ค Bot Token + Chat ID ใน /tech ตั้งถูก
  • ลองกดปุ่ม Test Telegram ใน /tech

กล้องไม่ขึ้นภาพ / มืด

  • กล้องอาจต้องการ 1-3 รอบ retry ตอน cold boot — รอ ~10 วินาที
  • ปรับความสว่างใน /tech → Camera
  • เช็คสายแพ MIPI-CSI หลังเครื่อง ต่อแน่นหรือไม่

มื้อยาตั้งไม่ได้ ขึ้น "เวลาไม่ถูกต้อง"

  • เช็คว่าเวลาก่อน < เวลาหลัง
  • เช็คว่าห่างก่อน/หลังไม่เกิน 1 ชม.
  • เช็คว่าไม่มีมื้ออื่นตั้งเวลาเดียวกัน
